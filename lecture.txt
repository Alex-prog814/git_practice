3. Создать репозиторий
4. Инициализация и подключение
    git init - инициализируем пустой локальный репозиторий
    git remote add origin 'SSH' - привязка удаленного репозитория
    git remote -v - посмотреть привязку
    git remote remove origin - удаление привязки(можно добавить хоть сколько привязок, главное название должно быть уникальным)
5. Ветки, рассказать зачем нужно и как создать(но стоит помнить, что пока нет мастера могут быть ошибки поэтому стоит создать файл и отправить на мастер пункт 6)
    git branch -v - проверить какие ветки есть
    git branch 'branch-name' - создать ветку
    git checkout 'branch-name' - перейти на указанную ветку
6. Оправка данных
    git add . - упаковать все файлы
    git commit -m 'First commit' - маркируем коммит
    git push origin master - отправляем данные на ветку мастер
